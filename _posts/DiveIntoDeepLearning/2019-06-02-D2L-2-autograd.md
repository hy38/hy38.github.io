###### 3) In the control flow example where we calculate the derivative of d with respect to a, what would happen if we changed the variable a to a random vector or matrix. At this point, the result of the calculation f(a) is no longer a scalar. What happens to the result? How do we analyze this?

Changing $a$ to a random matrix, the result $d = f(a)$ comes to be a matrix having same size of $a$.

Analizing the $d = f(a)$ for a random matrix $a$, it is very similar when $a$ is a scalar value.

The `==` calculation results some of the elements to be `false`, but it is because of the floating point division. You can see it below.

###### 4) Redesign an example of finding the gradient of the control flow. Run and analyze the result.

```python
def f2(a):
    b = a * a - 20 * a
    if b.sum() > 10:
        c = b
    else:
        c = -100 * b
    return c
```

Since $d = -100a^2 + 200a$, then $d' = a.grad = -200a + 200$.

As a result, a.grad will have the value of c', d.

```python
a = np.random.normal(10)
print(a)
# Result :
# 10.241542

a.attach_grad()
with autograd.record():
    d = f2(a)
d.backward()
print(d)
# Result :
# 9994.166

print(a.grad)
# Result :
# -48.30835

print((d / a) - (100 *a))
# Result :
# -48.30835

print(a.grad == (d / a) - (100 *a))
# Result :
# True
```

Note that the `True` and `False` can be changed by floating point division.

###### 5) Let 𝑓(𝑥)=sin(𝑥). Plot 𝑓(𝑥) and 𝑑𝑓(𝑥)𝑑𝑥, where the latter is computed without exploiting that 𝑓′(𝑥)=cos(𝑥).

```python
%matplotlib inline
import d2l
from IPython import display
x = np.arange(0, 3, 0.1)

def f(x):
    return np.sin(x)

x.attach_grad()
with autograd.record():
    sin = f(x)

sin.backward()
print(x.grad)
# Result :
# [ 1.          0.9950042   0.9800666   0.9553365   0.921061    0.87758255
#   0.8253356   0.7648422   0.6967067   0.6216099   0.5403023   0.4535961
```

![sin_cos](https://user-images.githubusercontent.com/34434143/83609736-a391de00-a5b9-11ea-86a2-8807f734a326.png)

Moreover, the graph x.grad seems to be cosine. So, lets see if it is.

```python

print(x.grad == np.cos(x))
# Result :
# [ True  True  True  True  True  True  True  True  True  True  True  True
#   True  True  True  True  True  True  True  True  True  True  True  True
#   True  True  True  True  True  True]

print(sin ** 2 + x.grad ** 2)
# Result :
# [1.         1.         1.         1.0000001  1.         1.
#  1.         1.         1.         1.         0.99999994 1.
#  1.         1.         1.         1.         0.99999994 1.0000001

```

Yes, it is cosine.
